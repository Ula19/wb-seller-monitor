# wbBot — мониторинг продавцов Wildberries

Telegram-бот следит за ассортиментом выбранных продавцов WB через внутренний API
(без токена): часовой отчёт по всем товарам + мгновенные уведомления о новинках.

## Возможности
- Часовой отчёт по каждому магазину (Excel для магазинов от 100 товаров).
- Проверка новинок каждые N минут → моментальное уведомление с фото.
- Общий список магазинов для всех пользователей.
- Доступ по whitelist: владелец управляет пользователями командами.
- Хранение в PostgreSQL: магазины, товары, дата первого обнаружения, дата проверки.

## Управление
Основной способ — **inline-кнопки**: `/start` или `/menu` открывает меню с навигацией
(Магазины, Проверить сейчас, для админа — Пользователи и Статистика). Добавление
магазина и выдача доступа идут пошаговым диалогом, удаление — выбором из списка кнопками.

Команды тоже работают:
- **Пользователь:** `/menu`, `/listsellers`, `/checknow`
- **Владелец:** `/addseller <ID|ссылка>`, `/removeseller <ID>`, `/grant <id>`, `/revoke <id>`, `/users`, `/stats`

`supplier_id` — число из ссылки магазина `wildberries.ru/seller/<ID>`.

## Запуск (Docker)
```bash
cp .env.example .env
# заполнить BOT_TOKEN и OWNER_ID
docker compose up -d --build
```

## Запуск локально
```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
# поднять Postgres и прописать DATABASE_URL в .env
python -m app.main
```

## Важно
- **Гео:** внутренний API WB доступен только с IP РФ/СНГ. VPS берите в СНГ
  (РФ/UZ/KZ/BY), иначе нужен российский резидентный/мобильный прокси.
- **Лимиты:** бот троттлит запросы (один воркер, задержки с jitter, backoff на 429),
  чтобы не словить бан по IP. Не уменьшайте `REQUEST_MIN_DELAY` без необходимости.
- Эндпоинты WB неофициальные (версии путей вида `v4` периодически меняются).

## Архитектура
```
app/
  config.py            настройки из .env
  db/      base, models (User/Seller/Product/Notified), repo
  wb/      client (httpx+троттлинг), parser (нормализация, basket-хосты)
  services/ monitor (синхронизация, рассылка, jobs), reporting (тексты, Excel, фото)
  bot/     access, middlewares, handlers (common/user/admin)
  main.py  точка входа: БД + бот + планировщик
```
